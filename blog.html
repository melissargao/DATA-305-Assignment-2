Building My Personal Website with an AI Assistant

By Melissa Gao | Published on March 11, 2026

In my DATA 305 Problem Solving with GenAI class, I was tasked with creating a personal website using an AI assistant. Completing this assignment gave me a much clearer understanding of what it takes to build and publish a website from scratch. The goal

Choosing a Site Tool

One of the first things I did was compare different tools I could use for the site. I looked at plain HTML/CSS, Jekyll, and Astro. Plain HTML/CSS appealed to me because it offered the most control and had the simplest setup. I liked the idea of working directly with the structure and styling of the site without having to learn a framework or deal with a complicated build process. Jekyll was also a strong option because it has native GitHub Pages support and a large template ecosystem, which would make deployment easier. Astro seemed attractive because it is modern and content-focused, but it also introduced more complexity than I felt I needed for this assignment. The instructions specifically encouraged us to consider ease of getting started, customization, deployment workflow, and how well an AI assistant could support each option, which became the criteria I used in my comparison. In the end, I chose plain HTML/CSS because it matched my goals for the project. Since this was a personal site with only a few pages, I felt that a lightweight approach made the most sense. I wanted to understand the core structure of the site instead of relying too heavily on a framework. I also thought plain HTML/CSS would be easier to troubleshoot with AI, since the code is more direct and the output is easier to inspect. That decision ended up shaping the rest of the project. It kept the site simple, but it also meant I had to be more intentional about organization, consistency, and responsiveness across pages.

Using AI for Starter Code

As I started on my website, AI became most helpful for generating starter code. I used it to draft page structures, navigation layouts, and section content, especially when I wanted a starting poibt for my homepage and blog post. It was also useful when I needed help turning vague design ideas into concrete code. For example, instead of staring at a blank file, I could ask for a simple responsive layout with a hero section, content cards, and a footer, then revise it to fit my own style. This made the process much faster than writing every line from scratch. At the same time, using AI effectively required much more specificity than I expected. One of the tips in the assignment was that more specific prompts lead to better results, and I found that to be completely true. When I asked broad questions, the responses were often too generic or included design choices that did not fit what I wanted. But when I gave detailed instructions about style, structure, tone, and content, the outputs became much more useful. I learned that AI works best as a collaborator when I already have a clear idea of the direction I want to go.

Building Process

Once I had chosen a tool and settled on a general design direction, building the actual pages felt manageable. I liked seeing the site slowly take shape, especially when the homepage, resume page, and blog started to feel visually connected. Getting the navigation to work across pages and making the design feel cohesive were satisfying milestones. I also appreciated that GitHub Pages made deployment accessible. Publishing the site gave the project a different feeling than a normal assignment because it became a real, public-facing product instead of something that only existed on my computer. I definitely faced several challenges along the way. Sometimes the AI-generated code was not as polished s it first appeared. It technically worked, but the spacing and hierarchy was awkward, so I learned pretty quickly that "working" code is not the same thing as good code or design. Deployment also came with a learning curve. Even though GitHub Pages is designed to make static site hosting easier, I still had to make sure my files were named correctly, linked correctly, and uploaded in the right structure.

Conclusion

The biggest thing I learned from this assignment was how to think about AI as a development partner rather than a replacement for my own judgment. AI was extremely helpful for generating ideas, writing starter code, and troubleshooting specific issues, but it was not reliable enough to use without careful review. I still had to decide what made sense, what matched the assignment requirements, and what fit the kind of site I wanted to create. In that way, AI made me more responsible, not less. It sped up the process, but it also required me to pay close attention and edit actively. By the end of the project, I had not only built a personal site, but also developed a better process for solving open-ended technical problems.
